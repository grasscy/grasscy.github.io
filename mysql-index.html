小记之mysql索引

发表于 2022-03-15 | 分类于 db ， mysql | 评论数：

前言

一些关于 mysql 索引的知识点。

导航

b 树与 b+树

  • B 树的所有节点既存放键(key) 也存放 数据(data)，而 B+树只有叶子节点存放 key 和 data，其他内节点只存放 key。
  • B 树的叶子节点都是独立的;B+树的叶子节点有一条引用链指向与它相邻的叶子节点。
  • B 树的检索的过程相当于对范围内的每个节点的关键字做二分查找，可能还没有到达叶子节点，检索就结束了。而 B+树的检索效率就很稳定了，任何查找都是从根节点到叶子节点的过程，叶子节点的顺序检索很明显。

聚簇索引与非聚簇索引

  • 聚簇索引：索引对应的data即是数据
    • 依赖有序数据，无序数据插入或查找的速度慢
    • 更新代价大
  • 非聚簇索引：索引对应的data不是直接的数据，需要继续查找（指针，主键……）
    • 依赖有序数据
    • 可能会二次查询（回表）
      • 也可能不回表，比如查索引本身值或主键

        覆盖索引

  • 一个索引包含（或者说覆盖）所有需要查询的字段的值

myisam 与 innodb 均采用 b+树

  • myisam:非聚簇索引

    MyISAM 引擎中，索引文件和数据文件分离，B+Tree 叶节点的 data 域存放的是数据记录的地址。

  • innodb:聚簇索引

    InnoDB 引擎中，其数据文件本身就是索引文件，其表数据文件本身就是按 B+Tree 组织的一个索引结构，树的叶节点 data 域保存了完整的数据记录。

主键索引与二级索引（辅助索引）

  • 主键索引：聚簇索引
  • 二级索引：非聚簇索引
    • 唯一索引
    • 普通索引
    • 前缀索引：只适用于字符串
    • 全文索引：大文本数据中的关键字信息

创建索引的注意事项

  • 不为null：数据库难优化
  • 频繁查询、作为条件查询、排序、频繁用于连接
  • 慎重：频繁更新
  • 联合索引>单列索引：空间
  • 扩展索引>创建索引
  • 字符串：前缀索引>普通索引
