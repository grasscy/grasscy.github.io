Hexo+Nginx+Docker搭建记录

Posted on 2018-10-20 | In other

本想搭建一个动态博客，可是考虑了自己的需求之后，发现自己需要的仅仅是一个拥有支持markdown、展示文章这两个基本功能的网站，因此最终选择了静态博客，也是近来流行的搭配：Hexo+next主题。

Hexo和next的搭建都很顺利，可是随着使用，渐渐产生了一些想法：

  • 采用腾讯云主机部署，如果要迁移可能有些麻烦
  • 每次hexo generate，hexo server还是有些繁琐，能不能一条命令即可？
  • hexo似乎不支持https？

Hexo支持git部署，将hexo部署到git上看起来是个不错的选择。github pages甚至已经支持了自定义域名使用https协议，关于hexo+github pages的教程也有大把。看似一切都很简单美好，自己部署时却出现了问题：

  • 访问速度。这虽然有点慢，暂时也没多大事。
  • github pages的文档上，DNS解析的A记录一共需配置4条ip，自己的腾讯云套餐只支持2个负载均衡。可能是由于这个原因，引发了下一个不可忽视的问题。
  • http://域名 无法访问。www.和https:都可以访问，而首次采用http:访问却不行。只有通过www./https访问之后，才可以通过http访问。

第三点我仔仔细细看了官方文档、网上各种教程，都没有找出原因和解决方法。so，hexo+github pages方案对我来说，行不通。

于是采取了另一办法，hexo+Nginx。nginx虽只闻其名不知其用，但也意识到，为了日后方便，还是用docker为好。因此，磕磕绊绊的完成dockerfile、docker-compose.yml、nginx的配置文件blog.conf之后，通过http终于可以正常访问了。

下一步就是配置https了z，没想到自己在这却卡了很久：无论是http跳转至https、还是直接通过https访问，浏览器都显示“连接超时”，调试窗口的network中一直是peeding状态。

反反复复改了多次blog.conf，最后找到的真实原因却很简单：安全组的443端口没开放，emmm…

不过最终还是完成了目标：hexo g -d，一条命令即可。
